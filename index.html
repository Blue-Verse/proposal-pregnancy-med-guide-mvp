핵심 요구사항 분석

  • If-Then 조건부 판정 알고리즘 — 엑셀 기반의 약사 노하우를 시스템화하여, 임신 주수·모유 수유 여부에 따라 동일 약물의 상이한 위험도와 복약 가이드를 자동 산출
  • 약물 데이터베이스 구축 — 엑셀 데이터의 일괄 업로드(Data Import) 및 관리자의 수동 등록·수정·삭제 지원
  • 주수별 예외 처리 — 특정 주수 이상/이하 조건에 따른 분기 처리로 정밀한 결과값 도출
  • OCR 약 봉투 인식 (선택) — 스마트폰 카메라를 활용한 처방전/약 봉투 텍스트 자동 인식
  • 관리자 웹(Admin) — 회원 관리, 약물 DB 관리, 엑셀 임포트, CRUD 기능
  • iOS/Android 동시 지원 — 하이브리드 앱 1종으로 양 플랫폼 커버

제안 포인트

1. 헬스케어 도메인 실전 경험

시니어 주간보호센터 투약 관리 플랫폼을 6개월간 풀스택으로 구축한 경험이 있습니다. 투약의뢰서, 전자서명, 건강 데이터 분석 등 의료·복약 도메인의 민감 정보 처리와 법적 요구사항을 숙지하고 있어, 임산부 약물 위험도 판정 시스템에 즉시 적용 가능합니다.

2. If-Then 조건부 알고리즘 구현 역량

사주(四柱) 분석 앱에서 복잡한 조건부 로직을 시스템화한 경험이 있습니다. 생년월일·시간에 따라 상이한 결과를 도출하는 구조가 임신 주수·모유 수유 여부에 따른 약물 위험도 판정과 동일한 패턴이므로, 엑셀 기반 판정 로직의 디지털화를 효율적으로 수행할 수 있습니다.

3. OCR 모바일 연동 경험

디지털 명함 플랫폼에서 카메라 기반 OCR 스캐닝을 Flutter 앱에 성공적으로 구현한 경험이 있어, 약 봉투/처방전 텍스트 인식 기능의 기술적 리스크를 최소화할 수 있습니다.

4. 정부지원사업 산출물 대응

요구사항 정의서, ERD, 화면설계서, 완료 보고서 등 정부지원사업 제출용 산출물을 체계적으로 관리하며, 사업 일정에 맞춘 단계별 납품이 가능합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

임신 중이거나 모유 수유 중인 여성은 약물 복용 시 태아/영유아에 미치는 영향을 우려하여 복용을 기피하거나, 반대로 정보 부족으로 위험한 약물을 복용하는 경우가 발생합니다. 본 프로젝트는 약사의 전문 노하우를 엑셀 기반 If-Then 판정 로직으로 시스템화하여, 개인의 임신 주수·수유 여부에 맞춤화된 약물 위험도 및 복약 안심 가이드를 제공하는 것을 목표로 합니다.

복잡한 AI 기술 대신 명확하고 안정적인 조건부 시나리오 로직을 적용하여 초기 MVP 모델을 완성하고, 신뢰도 높은 텍스트 기반 정보를 제공합니다.

2. 사용자 정의

관리자 (약사/대표자)약물 DB 및 판정 알고리즘 관리약물 등록/수정/삭제, 엑셀 임포트, 회원 조회
자문단 (산부인과 전문의)의학적 자문 및 가이드 검수가이드 텍스트 검수 (시스템 외부)

3. 기능 요구사항

3.1 사용자 계정 및 정보 관리

  • [필수] 이메일 또는 간편 로그인(카카오/Google) 연동 회원가입
  • [필수] 프로필 입력: 임신 주수, 모유 수유 여부, 기본 건강 상태
  • [필수] 마이페이지: 개인 정보 및 현재 상태 업데이트

3.2 약물 위험도 판정 및 매칭 시스템

  • [필수] 약물 명칭 및 증상 텍스트 검색
  • [필수] 사용자 데이터(임신 주수 등)와 약물별 조건(If-Then) 대조하여 결과값 산출
  • [필수] 특정 주수 이상/이하 조건에 따른 상이한 결과값 도출 분기 처리
  • [필수] 결과 화면: 약물 위험도 + 텍스트 기반 안심 가이드 출력
  • [필수] 약사 노하우 반영: 엑셀 데이터로 정의된 상황별 세부 복약 지침 텍스트

3.3 OCR 약 봉투 인식 (선택)

  • [선택] 스마트폰 카메라를 활용한 처방전/약 봉투 텍스트 자동 인식
  • [선택] 인식된 약물명 자동 검색 연동

3.4 관리자 시스템 (Admin Web)

  • [필수] 회원 목록 조회 및 상태 정보 모니터링
  • [필수] 엑셀 데이터 기반 약물 정보 일괄 업로드 (Data Import)
  • [필수] 신규 약물 등록: 약물 명칭, 주수별 If-Then 조건, 가이드 텍스트 수동 등록
  • [필수] 약물 정보 수정 및 삭제

5. 기술 제약사항

  • MVP 단계에서는 복잡한 AI 기술 대신 If-Then 조건부 로직만 적용
  • 엑셀 기반 판정 알고리즘의 정확한 디지털화가 핵심 — 약사(대표자)와 긴밀한 협의 필요
  • 의료 정보 특성상 결과 화면에 "전문의 상담 권장" 안내 문구 필수 포함
  • OCR 인식률은 인쇄 품질에 따라 차이 — 수동 입력 대안 필수 제공

6. 범위 외 사항 (Out of Scope)

  • 실시간 약사/의사 상담 기능
  • AI/ML 기반 자동 판정 (2차 고도화 시 검토)
  • 전자 처방전 연동 (의료기관 시스템 연계)
  • 다국어 지원 (MVP는 한국어만)
  • 결제/구독 시스템

커넥틴 — 디지털 명함 & 네트워킹 플랫폼

B2B 플랫폼 · 3개월

프로젝트 개요

물리적 명함 교환의 한계를 해결하기 위해 OCR 스캐닝, E2E 암호화 메시징, BLE/GPS 근거리 탐색을 결합한 디지털 명함 & 네트워킹 플랫폼. 12개 마이크로서비스, 63 API 엔드포인트.

핵심 기능

  • OCR 명함 스캐닝 — 카메라로 물리적 명함 촬영 시 자동 디지털 변환
  • Signal Protocol E2E 암호화 메시징 — 금융급 보안 수준
  • BLE + GPS 근거리 탐색 — 주변 사용자 실시간 발견
  • 관리자 패널 — 역할 기반 권한, 대량 사용자 관리

본 프로젝트와의 유사점

OCR 카메라 인식 + 매칭 알고리즘이 핵심 유사점입니다. 명함 OCR 스캐닝 → 텍스트 자동 인식 → 데이터 매칭의 흐름이 약 봉투 OCR → 약물명 인식 → If-Then 판정과 동일한 패턴입니다. Flutter 크로스플랫폼 구현 경험과 민감 정보 보안 처리도 직접 활용됩니다.

운세 앱 — 사주 기반 맞춤형 분석 앱

B2C 앱 · 3개월

프로젝트 개요

사주(四柱) 기반 맞춤형 운세·행운의 번호·궁합 추천 모바일 앱. Flutter 단일 코드베이스로 iOS + Android 동시 출시, 기획부터 출시까지 3개월 풀패키지 딜리버리.

핵심 기능

  • 생년월일/시간 입력 기반 사주 분석 — 복잡한 조건부 로직 시스템화
  • 사주 맞춤형 행운의 번호 추천 — 조건별 결과값 분기 처리
